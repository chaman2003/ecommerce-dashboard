Abstract
The Big Data Movie Analytics Dashboard is a full-stack, cloud-native web platform that ingests, stores, analyzes, and visualizes information about 1,001+ curated and procedurally generated movies. The solution employs Node.js 18+, Express 4.18, MongoDB Atlas, and Mongoose 8.0 on the backend, while the frontend leverages React 18.2, Material UI 5.14, Recharts 2.10, and Framer Motion 10.16 to deliver a responsive, neon-themed Vision UI experience with glass-morphism design patterns. Real movie metadata, multi-lingual coverage (20+ languages), and aggregation-driven insights enable intelligent filtering, recommendation, and storytelling features. The platform demonstrates advanced big data competencies including volume (1,000+ records), velocity (100 movies/batch, 390+ movies/second ingestion), variety (20+ countries, 19 genres), and veracity (curated datasets with validation). Live deployment on Vercel showcases serverless architecture optimization, connection pooling for MongoDB, and CI/CD integration.

- Vercel serverless functions are stateless
- Creating new MongoDB connection per request adds 200-300ms overhead

- Connection caching at module level
- Module variables persist across warm invocations
- Check connection state before creating new connection
- Reuse cached connection when ready

- Cold start: 500ms â†’ 200ms (60% reduction)

11. CHALLENGES AND MITIGATIONS
================================================================================

11.1 Challenge: Duplicate Title Handling
Problem: Generating 941 procedural movies with unique titles difficult
Mitigation: Incremental title generation with counters, pre-check against database
Result: 0% duplicate rate, 100% dataset uniqueness

11.2 Challenge: Slow MongoDB Connection on Vercel
Problem: New connection per request adds 200-300ms overhead
Mitigation: Connection caching at module level with state checking
Result: 60% reduction in cold start time

11.3 Challenge: Image Loading and CORS Issues
Problem: TMDB/Unsplash image loading failures and cross-origin issues
Mitigation: Eliminated images entirely, converted to text-only layout
Result: 100% reliability, faster page loads, cleaner design

11.4 Challenge: API 500 Errors in Production
Problem: MongoDB connection failures, timeout errors, poor diagnostics
Mitigation: Enhanced error handling, connection pooling, timeout config, health checks
Result: Reliable API with clear error messages

11.5 Challenge: Responsive Design Complexity
Problem: Multiple breakpoints, chart resizing, different mobile behavior
Mitigation: MUI responsive Grid, media queries, extensive testing
Result: Smooth experience across all screen sizes

11.6 Challenge: Performance Optimization
Problem: Bundle size, loading speed, unnecessary re-renders
Mitigation: Code splitting, lazy loading, memoization, debouncing
Result: Fast initial load, smooth interactions

11.7 Challenge: Multi-Parameter Filtering Logic
Problem: Complex query construction, ensuring all combinations work
Mitigation: Filter options endpoint, flexible query builder, extensive testing
Result: Efficient filtering, <100ms response times
